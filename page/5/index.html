JVM内存区域的划分 简单描述 -海子

###为什么要分堆和栈？
系统将程序对内存的使用分作堆和栈是出于对内存管理上的需要。栈用于保存一些临时性的局部变量，而堆则用于保存生命期较长，尤其是较大块的数据。

如此设计，背后的逻辑是：大多数局部变量都用以存储一些较小的数据，而且大都为临时性数据，因此在栈上分配和回收效率比较高。定义并使用这些局部变量的函数，在调用时也多为顺序执行，因此符合栈结构的线性特点。而对于较大块的数据，使用栈的代价往往比较高。因此即便是局部变量，如果¡大块数据，一般也都是定义在堆上的，并且使用完之后显式地进行销毁。设想，如果内存只有栈这一种结构，那么由于栈结构仅能在栈顶（或栈底）存取的特性，那么对大块与小块数据相互间杂的内存，访问的代价将是十分高昂的。相反，如果仅有堆，那就意味着，即使对于哪怕一个int型变量的定义和使用之后，也要显式地销毁这个变量所占用的内存，而这无疑又给程序员带来了额外的负担。总结一下，栈和堆的交互使用，是设计上对于易用性和高效性的一个折中，这既方便了内存的管理，降低了对程序员的要求，同时也增加了内存访问的效率。

Java8

流分为三种操作 。 中间操作，短操作， 结束操作。 一个流执行了一次结束操作之后这个流就不可以再次使用了。

流的操作是惰性的，它是根据判断，最后综合一起做，中间过程并没有一遍一遍的进行循环。

MAP :
ArrayList E = Lists.newArrayList(“A”, “B”, “C”, “D”);
List collect = E.stream().map(p -> p.toLowerCase()).collect(Collectors.toList());
或者
List collect = E.stream().map(String::toLowerCase()).collect(Collectors.toList());

Filter 和 foreach
ArrayList integers = Lists.newArrayList(1, 2, 3, 4, 5, 6, 7);
integers.stream().filter(a -> a >= 4 ).forEach(a -> System.out.println(a));

Groupby:
List persons = new ArrayList<>();
persons.add(new Person(1, “quanju”)) ;
persons.add(new Person(2, “jiahong”)) ;
persons.add(new Person(1, “pengfei”)) ;
persons.add(new Person(2, “chenshuai”)) ;
Map<Integer, List> collect1 = persons.stream().collect(Collectors.groupingBy(Person::getAge));
Iterator<Map.Entry<Integer, List>> iterator = collect1.entrySet().iterator();
while (iterator.hasNext()) {
Map.Entry<Integer, List> next = iterator.next();
System.out.println(“Age : “ + next.getKey() + “ Name” + next.getValue().toString());
}

欢迎使用 MWeb

MWeb 是专业的 Markdown 写作、记笔记、静态博客生成软件，目前已支持 Mac，iPad 和 iPhone。MWeb 有以下特色：

软件本身：

  • 使用原生的 macOS 技术打造，追求与系统的完美结合。
  • 原则上，首先是追求界面简洁和高性能，然后才是强大易用，功能全面。

Markdown 语法：

  • 使用 Github Flavored Markdown 语法，简称 GFM 语法。
  • 支持表格、TOC、LaTeX、代码块、任务列表、脚注等。
  • 画图库支持 mermaid, viz, echarts, plantuml, sequence, flow。

Markdown 辅助：

  • 支持截图并粘贴、复制并粘贴、拖拽等方式插入图片并直接显示在编辑器内。
  • 在兼容 Markdown 语法的情况下支持设置图片宽度。
  • 好用的表格插入和 LaTeX 书写辅助。

Markdown 输出：

  • 支持导出为图片、HTML、Epub、PDF、RTF、Docx。
  • 支持发布到 Wordrpess、支持 Metaweblog API 协议的服务、Wordpress.com、印象笔记（Evernote）、Blogger、Medium、Tumblr。
  • 图片上传服务（图床）支持 Google Photos、Imgur、七牛云、又拍云和自定义的图床服务。

Markdown 笔记：

  • 强大的文档库支持分类树和标签管理文档，文档可归类于多个分类，可以把分类整个导出为 Epub、PDF 和生成静态网站。非常合适用于笔记、个人知识收集、管理和输出。
  • 快速笔记：随意增加笔记及图片等素材，支持以天为单位把增加的素材组合在一个文档内，方便整理及记录历史收集情况。
  • 快速搜索：目前已支持全局快捷键调出搜索。

外部 Markdown 文档：

  • 外部 Markdown 文档使用外部模式管理。外部模式使用目录树的方式编辑和管理你的 markdown 文档，还能很好的支持 gitbook、JekyII、hexo 等等编辑和图片插入。

MWeb 文档：

如果要更详细了解 MWeb，建议你一定要去看一下官网的帮助文档，网址为：https://zh.mweb.im/help.html。我们建议在使用文档库之前，一定要阅读一下 MWeb 文档库详细介绍 这篇文章，以便更好的使用文档库。
