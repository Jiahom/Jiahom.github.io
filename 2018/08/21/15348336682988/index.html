#Java 动态代理机制
通过工具，可以在运行时动态的生成Java字节码对象，在生成这个字节码对象中加入我们自己需要增加的逻辑。
常用生成字节码工具
ASM

ASM 是一个 Java 字节码操控框架。它能够以二进制形式修改已有类或者动态生成类。ASM 可以直接产生二进制 class 文件，也可以在类被加载入 Java 虚拟机之前动态改变类行为。ASM 从类文件中读入信息后，能够改变类行为，分析类信息，甚至能够根据用户要求生成新类。
Javassist

Javassist是一个开源的分析、编辑和创建Java字节码的类库。是由东京工业大学的数学和计算机科学系的 Shigeru Chiba （千叶 滋）所创建的。它已加入了开放源代码JBoss 应用服务器项目,通过使用Javassist对字节码操作为JBoss实现动态AOP框架。javassist是jboss的一个子项目，其主要的优点，在于简单，而且快速。直接使用java编码的形式，而不需要了解虚拟机指令，就能动态改变类的结构，或者动态生成类

最开始的动态代理 （复杂类型）

最初的动态代理就是通过字节码生成工具，在每次调用目标类的时候动态的生成一个代理类，这个代理类实现了目标类的所有方法，并且在方法上额外加入了自己的代码逻辑。

缺点：这种方法增加了代码量，使得逻辑更加复杂，不容易实现。并且每个生成的代理类都包含了大量的业务代码，并且封装性也不够，完全不具有可拓展性和通用性。如果某个代理类的一些业务逻辑非常复杂，上述的动态创建代理的方式是非常不可取的！

JDK模式动态代理 实现相同接口

针对最初的代理模式的缺点，我们引入了 InvocationHandler角色

InvocationHandler角色中，包含了一个目标对象，而InvocationHandler角色又被包含在Proxy对象中，当Proxy对象调用目标方法的时候，不是通过自身去调用，而是通过调用InvocationHandler去调用目标方法。
InvocationHandler角色的实现类是我们自己实现的，我们重写它的Invoke方法来增加我们自己的方法逻辑（这样的话我们不是只能增加一种我们自己的方法逻辑？）。
